Шаг 1
Настройка и развертывание ReportPortal
В следующем руководстве описывается развертывание на Linux/Mac/Windows.

1. Загрузите последнюю версию ReportPortal Docker compose file с GitHub . Вы можете сделать это, выполнив следующую команду:

curl -LO https://raw.githubusercontent.com/reportportal/reportportal/master/docker-compose.yml
Убедитесь, что вы переопределили переменную среды службы UAT. RP_INITIAL_ADMIN_PASSWORD

2. Запустите приложение с помощью следующей команды:

docker-compose -p reportportal up -d --force-recreate
Где:

-p reportportal— добавляет префикс проекта 'reportportal' ко всем контейнерам

вверх— создает и запускает контейнеры

-д— режим демона

--force-recreate— воссоздает контейнеры

Полезные команды
логи docker-compose— показывает логи со всех контейнеров
логи докера <имя_контейнера>— показывает логи из выбранного контейнера
docker ps -a | grep "reportportal_" | awk '{print $1}' | xargs
docker rm -f
— удаляет все контейнеры ReportPortal
Более подробное руководство по установке можно найти по ссылке .

Шаг 2
Запустить ReportPortal
Если вы развертываете ReportPortal на отдельном хосте, используйте:

http://IP_ADDRESS:8080
В противном случае откройте ReportPortal и войдите в систему.

Важный!
ReportPortal готов к входу, когда в нижней части страницы входа вы увидите список служб (служба API, служба заданий, служба авторизации, пользовательский интерфейс службы).
Используйте следующие учетные данные:
Для доступа пользователя:
Авторизоваться:default
Пароль:1q2w3e
Для доступа администратора:
Авторизоваться:superadmin
Пароль:erebus
Пожалуйста, измените пароль администратора в целях безопасности.
